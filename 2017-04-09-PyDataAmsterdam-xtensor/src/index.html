Data structures for data science

State of the art

The three main languages of data science, Python, R and Julia all have concepts of

  • N-D arrays / tensors
  • Data frames

With slightly different semantics and in-memory representation.

And so is the case for frameworks such as Spark, tensorflow.

You've got

  • a cool C++ package to call on numpy array
  • code that cannot be easily vectorized (i.e. loops)

Lazy evaluation

res does not hold any value, nothing is evaluated ...


                                    xarray<double> a, b, c;
                                    init_tensor(a, b, c);
                                    auto res = sqrt(cos(a) + sin(b)) * c;

... until assigned to a container ...


                                xarray<double> d = res;
                                // or
                                xarray<double> d = sqrt(cos(a) + sin(b)) * c;

... or upon access


                                    // Assuming res is a 2D tensor
                                    double d = res(4, 2);

So what is res ?

res is an expression

Expression system

Client code


                                    xarray<double> a, b, c;
                                    init_tensor(a, b, c);
                                    xarray<double> res = sqrt(cos(a) + sin(b)) * c;

Traditional implementation


                                    xarray<double> tmp1 = cos(a);
                                    xarray<double> tmp2 = sin(b);
                                    xarray<double> tmp3 = tmp1 + tmp2;
                                    xarray<double> tmp4 = sqrt(tmp3);
                                    xarray<double> res = tmp4 * c;

xtensor implementation


                                    xarray<double> res(broadcast_shape(a, b, c));
                                    for(size_t i = 0; i < res.shape()[0]; ++i)
                                        res(i) = sqrt(cos(a(i)) + sin(b(i))) * c(i);

Client code
