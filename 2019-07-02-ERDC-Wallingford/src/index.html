The Jupyter Kernel Protocol

  • A well-specified protocol built upon web standards
  • Implemented for more than 40 languages

Implementations of the protocol

  • Some kernels have their own implementation of the protocol
    • Python (ipykernel)
    • R (IRkernel)
    • Julia (IJulia)
  • Many others make use of the wrapper kernel approach
    • The Octave and Matlab kernels

On the wrapper kernel approach

  • Dependency on the Python runtime
  • The wrapped interpreter generally does not have a Python API.
    • Typical approach is to capture text output from the interpreter!
    • This makes widgets and rich display impossible

In fact, a kernel is merely an executable implementing an inter-process communication protocol.

Non-python kernels should not require the Python runtime.

 

  • A modern C++ implementation of the Jupyter protocol
  • Enabling kernel authors to create new kernels without having to deal with the details of the Jupyter protocol.
  • Implements the widgets protocol

The Lazy Tensor Algebra Expression System

What is xtensor?

  • A C++ template library for multi-dimensional array manipulation

    • Followings the idioms of the C++ STL

      (iterator pairs, clear value semantics)

    • But also an API similar to that of numpy

What is xtensor?

  • Python bindings to enable xtensor APIs on numpy arrays.

  • Julia bindings to enable xtensor APIs on Julia arrays.

  • R bindings to enable xtensor APIs on R arrays.

  • Cookiecutter projects for authoring of Python, Julia, and R extensions.

What is xtensor?

  • BLAS bindings to enable BLAS operations on xtensor expressions.

  • SIMD acceleration kernels.

  • All open-source (BSD License).

Lazy evaluation

res does not hold any value, nothing is evaluated ...


                                    xarray<double> a, b, c;
                                    // ... initialization of a, b and c ...
                                    auto res = sqrt(cos(a) + sin(b)) * c;

... until assigned to a container ...


                                xarray<double> d = res;
                                // or
                                xarray<double> d = sqrt(cos(a) + sin(b)) * c;

... or upon access


                                    // Assuming res is a 2D tensor
                                    double d = res(4, 2);

res is an expression

Expression system

Client code


                                    xarray<double> a, b, c;
                                    // ... initialization of a, b and c ...
                                    xarray<double> res = sqrt(cos(a) + sin(b)) * c;

Traditional implementation


                                    xarray<double> tmp1 = cos(a);
                                    xarray<double> tmp2 = sin(b);
                                    xarray<double> tmp3 = tmp1 + tmp2;
                                    xarray<double> tmp4 = sqrt(tmp3);
                                    xarray<double> res = tmp4 * c;

xtensor generated implementation


                                    xarray<double> res(broadcast_shape(a, b, c));
                                    for(size_t i = 0; i < res.shape()[0]; ++i)
                                        res(i) = sqrt(cos(a(i)) + sin(b(i))) * c(i);

Expression system

Client code


                                    xarray<double> a, b, c;
                                    // ... initialization of a, b and c ...
                                    double res = (sqrt(cos(a) + sin(b)) * c)(4, 2);

The traditional implementation is equivalent to


                                    xarray<double> tmp1 = cos(a);
                                    xarray<double> tmp2 = sin(b);
                                    xarray<double> tmp3 = tmp1 + tmp2;
                                    xarray<double> tmp4 = sqrt(tmp3);
                                    xarray<double> tmp5 = tmp4 * c;
                                    double d = tmp5(4, 2);

The xtensor generated assembly is equivalent to


                                    double d = sqrt(cos(a(4, 2)) + sin(b(4, 2)))
                                               * c(4, 2);

Data structures for data science

Generalization of the dataframe data structure

  • A collection of N-D variables (instead of 1-D columns)
  • Each variable is a tensor with named dimensions and labeledd coordinates
  • Each named dimension corresponds to (instead of a single index for the dataframe)

A dataframe expression system

  • xframe is to xtensor what pandas and xarray are to numpy
  • Like with xtensor, operations on xframe are lazy and only evaluated upon access or assignment
  • The Data model of xframe is very close to that of PyData's xarray

Communicating with Notebooks

Notebook are often seen as an implementation of Donald Knuth's litterate programming. A program is given as an explanation of the program logic in a natural language alongside de source code.

Jupyter notebooks have proved invaluable

  • to provide a narative alongside code,
  • to walk a user through the logic of an algorithm.

Communicating with Notebooks

Notebook are not always ideal to address non-technical readers.

  • confused by code cells, or the need to run the notebook to see results.
  • following the order of the code in the narrative results in the most interesting content to be at the end of the document.

The security model for sharing notebooks is also a challenge.

  • Arbitrary code execution is the main feature of the plaform.
