The Jupyter Kernel Protocol

  • A well-specified protocol built upon web standards
  • Implemented for more than 40 languages

Implementations of the protocol

  • Some kernels have their own implementation of the protocol
    • Python (ipykernel)
    • R (IRkernel)
    • Julia (IJulia)
    • Many others make use of the wrapper kernel approach
    • The Octave and Matlab kernels

On the wrapper kernel approach

  • Dependency on the Python runtime
  • The wrapped interpreter generally does not have a Python API.
    • Typical approach is to capture text output from the interpreter!
    • This makes widgets and rich display impossible

In fact, a kernel is merely an executable implementing an inter-process communication protocol.

Non-python kernels should not require the Python runtime.

 

  • A modern C++ implementation of the Jupyter protocol
  • Enabling kernel authors to create new kernels without having to deal with the details of the Jupyter protocol.
  • Implements the widgets protocol

Communicating with Notebooks

Notebook are often seen as an implementation of Donald Knuth's litterate programming. A program is given as an explanation of the program logic in a natural language alongside de source code.

Jupyter notebooks have proved invaluable

  • to provide a narative alongside code,
  • to walk a user through the logic of an algorithm.

Communicating with Notebooks

Notebook are not always ideal to address non-technical readers.

  • confused by code cells, or the need to run the notebook to see results.
  • following the order of the code in the narrative results in the most interesting content to be at the end of the document.

The security model for sharing notebooks is also a challenge.

  • Arbitrary code execution is the main feature of the plaform.
