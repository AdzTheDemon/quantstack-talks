What is Jupyter?

A consistent set of tools meant to improve the workflow of engineers, scientists

  • from the exploratory phase of their work

  • to the communication of their result
  • including all the intermediary steps

The Jupyter Notebook

  • Interactive, browser-based computing environment

  • Exploratory data science, ML, visualization, analysis, stats
  • Reproducible document format:
    • Code
    • Narrative text (markdown)
    • Equations (LaTeX)
    • Images, visualizations
  • Over 50 programming languages
  • Everything open-source (BSD license)

Back to Python

What did Jupyter change for the Python community?

  • A large proportion of users of Python use it interactively
    • Not spending their time in an IDE writing Python files
    • But instead using an improved REPL: Jupyter or IPython
    • The REPL matters

The REPL matters

For a long time, the focus of the Python community was to improve the language, and less importance was given to the Interpreter.

The REPL could become a more central project:

  • Improving CPython APIs to facilitate building front-ends such as IPython and Jupyter
  • Including some of Jupyter features into CPython?

Jupyter rich mime type display

  • Could the rich mime type support of Jupyter be upstreamed into CPython?
  • IPython currently goes down to the python ast to determine whether to display an object or not.

Magics

Commands for Jupyter kernels that are not part of the language

  • IPython has an extensible "magics" framework, and over a hundred of such commands in the core library allowing to
    • make use of the shell to delete files, change current working directory
    • measure execution time, annotate code, completely switch context to another programming language

Communicating with Notebooks

Notebook are often seen as an implementation of Donald Knuth's litterate programming. A program is given as an explanation of the program logic in a natural language alongside de source code.

Jupyter notebooks have proved invaluable

  • to provide a narative alongside code,
  • to walk a user through the logic of an algorithm.

Communicating with Notebooks

Notebook are not always ideal to address non-technical readers.

  • confused by code cells, or the need to run the notebook to see results.
  • following the order of the code in the narrative results in the most interesting content to be at the end of the document.

The security model for sharing notebooks is also a challenge.

  • Arbitrary code execution is the main feature of the plaform.
